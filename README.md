# Birthday Board

## Overview

Birthday Board is a full-stack birthday management app built for a job assignment.

The app allows authenticated users to view a shared birthday board, see whose birthday is today, and add, edit, or delete birthday records.

## Tech Stack

* Frontend: React + TypeScript
* Backend: Node.js + Express + TypeScript
* Database: Prisma with a cloud-hosted SQL database on Railway
* Authentication: JWT

## Project Structure

```bash
Birthday Board/
├── client/   # React frontend
├── server/   # Express backend + Prisma
└── package.json   # Root setup scripts
```

## Prerequisites

Before running the project, make sure you have:

* Node.js 20+
* npm
* Git

## Quick Start

Clone the repository:

```bash
git clone <repo-url> "Birthday Board"
cd "Birthday Board"
```

From the project root, run:

```bash
npm install
npm run install:all
npm run setup
npm run dev
```

These commands will:

* install the root helper dependencies
* install backend and frontend dependencies
* automatically create `server/.env` with the evaluation environment variables
* generate the Prisma client
* run database migrations
* seed the Railway database with demo data
* start both the backend and frontend together

The backend runs on:

```text
http://localhost:4000
```

The frontend runs on:

```text
http://localhost:5173
```

## Available Root Scripts

From the project root, the following scripts are available:

```bash
npm run install:all
```

Installs dependencies for both the backend and frontend.

```bash
npm run setup
```

Creates `server/.env`, runs Prisma generate, runs migrations, and seeds the database.

```bash
npm run dev
```

Starts both the backend and frontend together.

```bash
npm run dev:server
```

Starts only the backend.

```bash
npm run dev:client
```

Starts only the frontend.

## Database Setup

The project uses a Railway cloud database through the `DATABASE_URL` generated by the setup script.

The setup command runs:

```bash
npm run setup
```

This creates:

* one demo user
* several birthday records
* at least one birthday record for today

This allows the app to run from a clean clone with demo data.

## Design Notes

The birthday board is shared/global by design.

Any authenticated user can view and manage the shared birthday list. This was an intentional design choice for this assignment.

## API Endpoints

### Auth

```http
POST /api/auth/register
POST /api/auth/login
```

### Birthdays

All birthday routes require a JWT token:

```http
Authorization: Bearer <token>
```

```http
GET    /api/birthdays/today
GET    /api/birthdays?page=<page>&limit=<limit>
POST   /api/birthdays
PUT    /api/birthdays/:id
DELETE /api/birthdays/:id
```

## Troubleshooting

### Server fails to start

Make sure you ran the setup command from the project root:

```bash
npm run setup
```

This command creates the `server/.env` file automatically and prepares Prisma.

Then run:

```bash
npm run dev
```

### Client cannot connect to backend

Make sure the backend is running on:

```text
http://localhost:4000
```

Also make sure the frontend is running on:

```text
http://localhost:5173
```

### Seed fails

Make sure:

* the Railway database is reachable
* the generated `server/.env` file exists
* migrations were run before seeding

Then run again from the project root:

```bash
npm run setup
```

### Login fails

Make sure the seed script was executed successfully.

Then log in with:

```text
Email: demo@birthdayboard.local
Password: DemoPass123!
```
